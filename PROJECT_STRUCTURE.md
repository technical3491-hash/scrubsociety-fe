## Path Aliases

Configured in `tsconfig.json`:

- `@/*` → `./src/*`
- `@/components/*` → `./src/components/*`
- `@/lib/*` → `./src/lib/*`
- `@/hooks/*` → `./src/hooks/*`
- `@/styles/*` → `./src/styles/*`
- `@/types/*` → `./src/types/*`
- `@/config/*` → `./src/config/*`

## Organization Principles

### 1. **App Router (`src/app/`)**
- Contains all route pages
- Route groups like `(auth)` organize related routes
- Each route folder can have its own `layout.tsx` and `page.tsx`

### 2. **Components (`src/components/`)**
- **`ui/`** - Primitive UI components (shadcn/ui)
- **`layout/`** - Layout components (Navbar, Footer, Sidebar)
- **`forms/`** - Form-specific components
- **`charts/`** - Chart/visualization components
- Root-level: Feature-specific components

### 3. **Features (`src/features/`)**
- Domain-specific modules (e.g., `users/`, `dashboard/`)
- Each feature can have its own components, hooks, services
- Useful for larger applications

### 4. **Shared Resources**
- **`hooks/`** - Reusable React hooks
- **`lib/`** - Utilities and API clients
- **`styles/`** - Global styles
- **`types/`** - TypeScript definitions
- **`config/`** - Configuration files
- **`context/`** - React contexts
- **`store/`** - State management

## File Naming Conventions

- **Components**: PascalCase (e.g., `Navbar.tsx`, `CaseCard.tsx`)
- **Hooks**: camelCase starting with `use` (e.g., `useMobile.tsx`)
- **Utilities**: camelCase (e.g., `utils.ts`, `queryClient.ts`)
- **Types**: camelCase or kebab-case (e.g., `index.d.ts`)
- **Config**: camelCase (e.g., `env.ts`, `routes.ts`)

## Best Practices

1. **Route Groups**: Use `(folderName)` to organize routes without affecting URLs
2. **Co-location**: Keep page-specific components near their routes when possible
3. **Reusability**: Put shared components in `src/components/`
4. **Type Safety**: Define types in `src/types/` or co-locate with features
5. **Environment**: Use `src/config/env.ts` for environment variables
6. **Routes**: Use `src/config/routes.ts` for route constants
